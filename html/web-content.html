Web content

Key principles

Each website has a source code that is delivered to the client (browser). It can be done via HTTP, via local file system etc. Then the browser processes document and renders the display to the user. Final appearance is a combination of source construction (by the author) and rendering capability (in web browser). All web documents use HTML – Hypertext Mark-up Language. Other embedded technologies enhance display and behaviour (CSS, JavaScript).

A web document typically has three concerns:

  • Content – the information conveyed by a page (HTML is used for this purpose).
  • Presentation – the appearance of a page (CSS is used for this purpose).
  • Behaviour – interactive or responsive functionality (JavaScript (client-side) and PHP, Perl, Java, Ruby etc. (server-side) are used for this purpose).

Separation of content from presentation is a good practice in web development and tells you for example, semantic markup versus stylistic instructions, heading level versus large font, strong emphasis versus bold, new paragraph versus line break plus consistency, accessibility, ease of maintenance etc.

HTML

HTML (Hypertext Markup Language) is a text-based approach to describing how content contained within an HTML file is structured. This markup tells a web browser how to display the text, images and other forms of multimedia on a webpage. HTML was originally designed to be very simple. As the web became popular and browsers developed, new features were added to subsequent versions (page structure, forms, images, tables, frames, support for CSS, support for dynamic scripting with JavaScript, accessibility features, standardised support for multimedia and embedded objects).

HTML originally only targeted at Web browsers on PCs and workstations:

  • Forgiving of syntax errors and variations in HTML markup (elements with and without end tags, uppercase and lowercase tag names etc.).
  • It requires more processing at browser than a markup language with stricter syntax.
  • Causes problems for constrained devices and for machine-oriented processing of markup.

All of these problems led to the development of XHTML 1.0. The syntax for HTML was tightened up using XML mark-up rules. XHTML uses the same tags and attributes as HTML 4.01 with differences in structure and syntax.

XML is a standardised, but flexible specification for creating markup languages it was derived directly from SGML (as was HTML) and designed for online data description and exchange.

XHTML is advantageous because it was designed for internationalization, accessibility, device independence, usability and document structuring. It also lead developers towards enforced separation of concerns. However it is not so good because of its complex evolution. It requires new/re-engineered renderers to benefit over existing HTML development. It is unwieldy in contemporary use cases.

The newest version of HTML is HTML5. It is the fifth and current major version of the HTML standard, and subsumes XHTML. It contains a lot of new functionalities and is widely used nowadays.

CSS

Plain webpages are dull. Additional meaning and aesthetics enhance (and influence) user experience. Cascading Style Sheets provide a central location in which information about what various fonts, foreground colors, background colors, italicization and emphasization should be applied to various HTML elements within a webpage. Cascading style sheets can also control how various parts of a page are laid out on it. This can be very useful when the same website needs to be presented on different devices or platforms.

In 1994, one major component that was still missing was a tool to style web pages. In this year, 3 days before Netscape was made available as a browser, Håkon Wium Lie published the first draft of the Cascading HTML Style Sheets proposal. The proposal took into account that neither the author nor the reader would have complete control of the style of the document, their styles needed to be combined. After a lot of debate, with Microsoft signaling they would support CSS in their browsers, it was selected by W3C as the recommended styling tool for the web. Today CSS is the main tool used for styling websites.

JavaScript

JavaScript is a scripting language with object oriented capabilities that executes in the Web browser on the client machine. The browser has a JavaScript interpreter and is the script execution environment. JavaScript is embedded in XHTML, i.e. the code to run arrives with the web page. JavaScript has C-style syntax (contains dynamic typing and runtime evaluation). The only connection to Java is in the name.

JavaScript is used for interaction with user behaviour (mouse movement, key presses etc.) and interaction with browser environment (browser driven events, browser-type driven behaviour).

In 1995, Marc Andreessen, founder of Netscape Communications, had the vision that the web needed a way to become more dynamic. The web needed a languge that could interact with the DOM. To achieve this, Brendan Eich was famously contracted by Netscape to design such a language on a very tight deadline. Eich created a quick concept using Java-based syntax. He then went on to incorporate a unique style of object-oriented programming. And eventually JavaScript - the most popular scripting language for the web - was born.
